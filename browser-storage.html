Browser Storage Types

Local Storage

Overview: Local Storage enables a website to save up to 5MB of data (or more with vendor prefixes) on the client-side where it will survive browser restarts.

Key Features:

  • Storage of data never expires
  • Can keep a big amount of data locally.

Ideal Use Cases:

  • Data caching for offline use in a PWA.
  • Storing anything from non-sensitive info like preferences.

Disadvantages:

  • Just for Strings which means you cannot save hard data directly, period.
  • No security; the data can be seen by any script on the same domain.

Cookies

Overview: Cookies are small (up to 4 KB) files pushed to the client's hard drive (we use them at Influence for session management).

Key Features:

  • Store small data that expires.
  • Automatically sent with each HTTP request.

Ideal Use Cases:

  • Persisting tokens for accuracy of authentication.
  • Log user activity and preferences.

Disadvantages:

  • Uploaded 4KB maximum.
  • Sent with every HTTP request, which is going to add heavy traffic on the network.
  • Security issues, especially if it is not properly encrypted (e.g., XSS).

Session Storage

Overview: Session Storage is similar to Local Storage but for the length of a session. Naturally, data is deleted when the tab/window is closed.

Key Features:

  • Session and single-use data.
  • Not shared between tabs.

Ideal Use Cases:

  • Storing transient data such as fields you filled in a form or session data.

Disadvantages:

  • When the tab/window is closed, data is lost.
  • Applies to one tab or window — not shared across tabs.

IndexedDB

Overview: IndexedDB is a high-level database for client-side storage of large key-value structured datasets. So it can manage hundreds of MBs to terabytes of data.

Key Features:

  • Holds massive structured data.
  • Data that is not flaky and lives forever.

Ideal Use Cases:

  • Houses big data sets (i.e., Datasets the size of documents/files).
  • Add offline capabilities for web apps.

Disadvantages:

  • Harder to integrate than other types of storage.
  • For smaller projects and data storage requirements, it might be more than this simple solution.

Storage Choices – Choosing the Right Storage Solution

  • Local Storage — Use this for read-only, non-sensitive data but be aware that nothing is secret.
  • Cookies — Use cookies for session management or tracking but beware of the size and security concerns.
  • Session Storage — For short-term data that needs to be cleared on session end & limited to a single tab.
  • IndexedDB — Use for large/or structured data and offline support, however, it is easier to take care of.
